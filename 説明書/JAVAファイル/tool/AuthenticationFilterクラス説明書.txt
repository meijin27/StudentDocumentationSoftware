クラス説明書: AuthenticationFilter

1. 概要
このクラスは、ウェブアプリケーションのページ遷移を制御するためのフィルターです。
セッションの保持状態に応じて遷移可能なページを制御します。
例えば、未ログイン状態ではメインメニューにアクセスできません。

2. コード詳細
2.1. メソッド
doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
このメソッドは、セッションに応じてページ遷移を処理します。
リクエストのURLから、対応するセッションの保持状態に応じてページ遷移を制御します。
正規のページ遷移が行われなかった場合はリクエスト通りのURLに遷移させません。

init(FilterConfig fConfig) 
処理なし

destroy()
処理なし

3. 使用方法
アクセスするURLに応じて自動的に適用されます。
受け取ったリクエストのURLから、遷移すべきページを特定し、そのページに遷移させます。

4. 特徴
ユーザーによる誤操作を防止するため適切なページ遷移を実現します。
セッションにページ遷移に必要な情報が格納されているかを確認し、格納されていればリクエスト通りにページ遷移します。
格納されていなければ適切なページに強制遷移します。
例えば、未ログイン状態でメインページにアクセスしてもログイン画面に遷移します。
